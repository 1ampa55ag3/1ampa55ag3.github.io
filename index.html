Computer Security

OSI Security Architecture

Aspects of Security

  • Security attack
  • Security mechanism: detect,prevent,recover
  • Security service

1️⃣ Security Attack

Passive Attack

Do not affect system resources , just eavesdropping & monitoring

  • Release of Message Contents
  • Traffic Analysis
    Why Difficult to detect
  • Message transmission apparently normal
  • Emphasis on prevention rather than detection=>Encryption

Active Attack

Try to alter system resources or affect their operation

  • Masquerade（伪装）
  • Replay=>use nonce to prevent
  • Modification of Messages
  • Denial of Service(DoS)
    Difficult to prevent
  • The goal is to detect and recover

2⃣️ Security Service

1.Enhance security of data processing systems and information transfers of an organization
2.Intended tp counter security attack
3.Using one or more security mechanisms
4.Often replicates functions normally associated with physical documentssignatures,dates...

X.800

  • Authentication=>assurance that communicating entity is the one claimed
  • Access Control=>prevention of the unauthorized use of a resource
  • Data Confidentiality=>protection of data from unauthorized disclosure
  • Data Integrity=>assurance that data received is as sent by an authorized entity
  • Non-Repudiation=>protection against denial by one of the parties in a communication
  • Availability=>resource accessible/usable

3️⃣ Security Mechanism

1.Designed to Detect,Prevent,Recover from a security attack
2.No single mechanism that will support all services required
3.One particular element underlies many use of the mechanisms in use:Cryptographic techniques
4.Hence our focus on this topic

Model for Network Security

Requirements of this model

  1. Design a suitable algorithm for the security transformation;
  2. Generate the secret information(keys) used by the algorithm;
  3. Develop methods to distribute and share the secret information;
  4. Specify a protocol enabling the principals to use the transformation and secret information for a security service.

Model for Network Access Security

Requirements of this model

  1. Select appropriate gatekeeper functions to identify users;
  2. Implement security controls to ensure only authorized users access designated information or resources.

Network Monitoring Methods

Traffic Information capture methods

  • Network TAPs
  • Port mirroring

Network TAPs

1.A passive splitting device implemented inline between a device of interest and the network.
2.A tap forwards all traffic including physical layer errors to an analysis device.
3.If it fails or loses power,traffic between the firewall and internal router is not affected.

Traffic Mirroring and SPAN

Enables the switch to copy frames of one or more ports to a Switch Port Analyzer(SPAN) port connected to an analysis device.

Network Security Monitoring Tools

  1. Protocol Analyzers=>Capture traffic like Wireshark,Tcpdump,etc..
  2. NetFlow=>Provides a complete audit trail of basic information about every IP flow forwarded on a device
  3. SIEM=>Security Information Event Management system
  4. SNMP=>Simple Network Management Protocol

Attacking the Foundation

IP Vulnerabilities

  1. ICMP Attacks

    ICMP was developed to carry diagnostic messages and to report error conditions when routes,hosts,and ports are unavailable.=>Ping of Death & ICMP Flood

  2. DoS and DDoS Attacks

    Prevent legitimate users from gaining access to websites,email,online accounts,and other services;
    DDoS=>Larger in magnitude than a DoS attack,attacks introduced new terms such as botnet,handler systems,and zombie computers.

  3. Address Spoofing

    A threat actor creates packets with false source IP address information to either hide the identity of the sender or to pose as another legitimate user,then can gain access to otherwise inaccessible data or circumvent security configurations.

  4. MITM(Man in the middle) Attacks
  5. Session Hijacking

    TCP and UDP Vulnerabilities

    TCP Provides Reliable Delivery,Flow Control,Stateful Communication.
    TCP attacks target expected protocol behaviors

  6. TCP SYN flood attack
  7. TCP reset attack
  8. TCP session hijacking

    UDP=>A simple protocol that provides the basic transport layer functions,Connectionless,Lack of encryption
    UDP protocol attacks target the lack of protocol behaviors

  9. UDP checksum attack
  10. UDP flood attack
  11. UDP DoS attacks

Attacking What We Do

IP Service 👇

ARP Vulnerabilities=>ARP Cache Poisoning

ARP cache poisoning attacks deliberately poison the cache of another computer with spoofed IP address to MAC address mappings.

DNS Attacks

  1. DNS Open Resolvers(public name servers)
  2. DNS Stealth Attacks
  3. DNS Shadowing Attacks=>Hijacked domains are used to create subdomains which are used to resolve to malicious web sites.
  4. DNS tunneling Attacks=>Hides malicious instructions inside DNS queries and responses.

    Use DNS tunneling place non-DNS traffic within DNS traffic to circumvents(规避) security solutions,and the different types of DNS records such as TXT,MX,SRV,NULL,A or CNAME are altered.

    DHCP Attack

    Could result in every host on the network communication with malicious DNS servers and gateways.A DHCP spoofing attack creates a rouge DHCP server to serve falsified information.

Enterprise Services👇

HTTP & HTTPS

  1. Malicious iFrames

    An iFrame allows a page from a different domain to be opened inline within the current page.It can be used to launch malicious code.

  2. HTTP 302 cushioning

    Allows a web page to redirect and open in a different URL.Can be used to redirected to malicious code.

  3. Domain shadowing

    Malicious web sites are created from subdomains created from a hijacked domain.

    Email

  4. Attachment-based attacks

    Email with malicious executable files attached.

  5. Email Spoofing

    Phishing(网络诱骗）attack where the message appears to come from a legitimate source.

  6. Spam email

    Unsolicited email with advertisements or malicious content.

  7. Open mail relay server

    Massive amount of spam and worms can be sent by misconfigured email servers.

  8. Homoglyphs

    Phishing scheme where text characters (hyperlinks) look similar to real text and links.

    Web-Exposed Databases

  9. Command injection attacks

    Insecure code and web applications allows OS commands to be injected into form fields or the address bar.

  10. XSS Cross-site scripting attacks

    Insecure server-side scripting where the input is not validated allows scripting commands to be inserted into user generated forms fields, like web page comments.This results in visitors being redirected to a malicious website with malware code.

  11. SQL injection attacks

    Insecure server-side scripting allows SQL commands to be inserted into form fields where the input is not validated.

  12. HTTP injection attacks

    Manipulation of html allows executable code to be injected through HTML div tags, etc.

第一讲 初识数据库系统

引言

数据库系统发展的三代演变：层次/网状数据库系统=>关系数据库系统=>新一代数据库系统。

第一章 绪论

1.1 数据库系统概述

（1）数据库的4个基本概念

01 数据（Data）

  • 定义：描述事物的符号记录
  • 种类：数字、文字、图形、音频···
  • 语义：即数据的含义，数据与其语义不可分
  • 解释：数据的形式不能完全表达其内容，需要语义+解释
  • 数据是有结构的：记录是计算机存储数据的一种格式或方法

02 数据库（Database，DB）

  • 定义：长期储存在计算机内、有组织的、可共享的大量数据的集合
  • 基本特征：数据按一定数据模型组织、描述和储存；可为各种用户共享、冗余度较小、易扩展；数据独立性较高。

03 数据库管理系统（Database Management System，DBMS）

  • 定义：位于用户应用与操作系统之间的一层数据管理软件（基础软件，大型复杂的软件系统）
  • 用途：科学地组织和存储数据、高效地维护和获取数据
  • 主要功能
    1. 数据定义功能
      1. 提供数据定义语言DDL；
      2. 定义数据库中的数据对象。
    2. 数据组织、存储和管理
      1. 分类组织、存储和管理各种数据
      2. 确定数据在存储级别上的结构和存取方式
      3. 实现数据之间的联系
      4. 提供多种存取方式提高存取效率
    3. 数据操纵功能
      1. 提供数据操纵语言（DML）
      2. 实现对数据库的基本操作（查询、插入、删除和修改）
    4. 数据库的事务管理和运行管理
      1. 数据的安全性、完整性、名用户对数据的并发使用
      2. 发生故障后的系统恢复数据库
      3. 由数据库管理系统统一管理和控制，保证事务正确运行
    5. 数据库的建立和维护功能
      1. 提供实用程序/工具，完成数据库数据批量装载，数据库转储，介质故障恢复，数据库的重组织和性能监视等
    6. 其它功能
      1. 数据库管理系统与网络中其它软件系统的通信
      2. 数据库管理系统系统之间的数据转换
      3. 异构数据库之间的互访和互操作

04 数据库系统（Database System，DBS）

  • 定义：是指在计算机系统中引入数据库后的系统构成。
  • 在不引起混淆的情况下常常把数据库系统简称为数据库
  • 构成
    • 数据库
    • 数据库管理系统（及其应用开发工具）
    • 应用程序
    • 数据库管理员（ DataBase Administrator，DBA）

（2）数据管理技术的产生和发展

什么是数据管理？

  • 对数据进行分类、组织、编码、存储、检索和维护
  • 数据处理和数据分析的中心问题

发展过程

  • 人工管理阶段（~50s）
  • 文件系统阶段（50s~60s）
  • 数据库系统阶段（60s~）

    （3）数据库系统的特点

  • 数据结构化
    • 数据的最小存取单位是数据项
    • 数据用数据模型描述，无需应用程序定义
  • 数据的共享性高，冗余度低且易扩充
    • 面向整个系统，可以被多个用户、应用共享
    • 避免数据之间的不相容性与不一致性
  • 数据独立性高
    • 逻辑独立性
    • 物理独立性
    • 一方（逻辑/物理结构）发生变化时，另一方（应用程序）不受影响**（由DBMS的二级映像功能实现）**
  • 数据由数据库管理系统统一管理和控制
    • 数据的安全性（Security）保护，保护数据以防止不合粪的使用造成的数据的泄密和破坏。
    • 数据的完整性（Integrity）检查，保证数据的正确性、有效性和相容性。
    • 并发控制（Concurrency Control），对多用户的并发操作加以控制和协调，防止相互干扰而得到错误的结果
    • 数据库恢复（Recovery），将数据库从错误状态恢复到某一已知的正确状态。

      1.2 数据模型

      （1）概念

  • 数据模型是对现实世界数据特征的抽象。
  • 数据模型是数据库系统的核心和基础。
  • 分为概念模型、逻辑模型和物理模型两大类。

（2）概念模型

  • 按照用户的观点对数据和信息建模，用于数据库设计
  • 对信息世界的建模
  • 数据库设计的有力工具
  • 数据库设计人员和用户交流的语言
  • 表示方法：E-R图（Entity-Relationship）

    （3）逻辑模型和物理模型

  • 逻辑模型主要包括网状模型、层次模型、关系模型、面向对象数据模型、对象关系数据模型、半结构化薮据模型等
  • 按计算机系统的观点对数据建模，用于DBMS实现
  • 物理模型是对数据最底层的抽象，描述数据在系统上的表示方式和存取方法

    （4）信息世界的基本概念

  1. 实体（Entity），客观存在并可相互区别的事物称为实体,可以是具体的人、事、物或抽象的概念
  2. 属性（Attribute），实体所具有的某一特性称为属性。一个实体可以由若干个属性来刻画
  3. 码（Key），唯一标识实体的属性集称为码
  4. 实体型（Entity Type），用实体名及其属性名集合来抽象和刻画同类实体称为实体型
  5. 实体集（Entity Set），同一类型实体的集合称为实体集
  6. 联系（ Relationship），现实世界中事物内部以及事物之间的联系在信息世界中反映为实体（型）内部的联系和实体（型）之间的联系
    • 实体内部的联系：是指组成实体的各属性之间的联系
    • 实体之间的联系：通常是指不同实体集之间的联系

      （5）数据模型的组成要素

      1）数据结构——描述系统的静态特性
  • 刻画数据模型性质的重要方面
  • 根据数据结构的类型命名数据模型（例：层次结构=>层次模型）
  • 描述数据库的组成对象——对象的类型、内容、性质
  • 描述对象之间的联系

2）数据操作——描述系统的动态特性

  • 对数据库中各种对象的实例允许执行的操作的集合，包括操作及有关的操作规则
  • 数据操作的类型：查询，更新（删除、修改、插入）
  • 数据操作的语言
    • 定义数据操作的确切含义、符号、优先级别
    • 实现数据操作的语言
      • 查询语言 —— Query Language
      • 更新语言 —— DML

3）数据的完整性约束条件

  • 一组完整性规则的集合
    • 完整性规则：给定的数据模型中数据及其联系所具有的制约和依存规则
    • 用以限定符合数据模型的数据库状态以及状态的变化，以保证数据的正确、有效和相容
  • 数据模型对完整性约束条件的定义
    • 反映和规定必须遵守的基本的通用的完整性约束条件
    • 提供定义完整性约束条件的机制，以反映具体应用所涉及的数据必须遵守的特定的语义约束条件
