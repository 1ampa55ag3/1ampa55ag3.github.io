Model for Network Security

Requirements of this model

  1. Design a suitable algorithm for the security transformation;
  2. Generate the secret information(keys) used by the algorithm;
  3. Develop methods to distribute and share the secret information;
  4. Specify a protocol enabling the principals to use the transformation and secret information for a security service.

Model for Network Access Security

Requirements of this model

  1. Select appropriate gatekeeper functions to identify users;
  2. Implement security controls to ensure only authorized users access designated information or resources.

Network Monitoring Methods

Traffic Information capture methods

  • Network TAPs
  • Port mirroring

Network TAPs

1.A passive splitting device implemented inline between a device of interest and the network.
2.A tap forwards all traffic including physical layer errors to an analysis device.
3.If it fails or loses power,traffic between the firewall and internal router is not affected.

Traffic Mirroring and SPAN

Enables the switch to copy frames of one or more ports to a Switch Port Analyzer(SPAN) port connected to an analysis device.

Network Security Monitoring Tools

  1. Protocol Analyzers=>Capture traffic like Wireshark,Tcpdump,etc..
  2. NetFlow=>Provides a complete audit trail of basic information about every IP flow forwarded on a device
  3. SIEM=>Security Information Event Management system
  4. SNMP=>Simple Network Management Protocol

Attacking the Foundation

IP Vulnerabilities

  1. ICMP Attacks

    ICMP was developed to carry diagnostic messages and to report error conditions when routes,hosts,and ports are unavailable.=>Ping of Death & ICMP Flood

  2. DoS and DDoS Attacks

    Prevent legitimate users from gaining access to websites,email,online accounts,and other services;
    DDoS=>Larger in magnitude than a DoS attack,attacks introduced new terms such as botnet,handler systems,and zombie computers.

  3. Address Spoofing

    A threat actor creates packets with false source IP address information to either hide the identity of the sender or to pose as another legitimate user,then can gain access to otherwise inaccessible data or circumvent security configurations.

  4. MITM(Man in the middle) Attacks
  5. Session Hijacking

    TCP and UDP Vulnerabilities

    TCP Provides Reliable Delivery,Flow Control,Stateful Communication.
    TCP attacks target expected protocol behaviors

  6. TCP SYN flood attack
  7. TCP reset attack
  8. TCP session hijacking

    UDP=>A simple protocol that provides the basic transport layer functions,Connectionless,Lack of encryption
    UDP protocol attacks target the lack of protocol behaviors

  9. UDP checksum attack
  10. UDP flood attack
  11. UDP DoS attacks

Attacking What We Do

ARP Vulnerabilities=>ARP Cache Poisoning

ARP cache poisoning attacks deliberately poison the cache of another computer with spoofed IP address to MAC address mappings.

DNS Attacks

  1. DNS Open Resolvers(public name servers)
  2. DNS Stealth Attacks
  3. DNS Shadowing Attacks=>Hijacked domains are used to create subdomains which are used to resolve to malicious web sites.
  4. DNS tunneling Attacks=>Hides malicious instructions inside DNS queries and responses.

    Use DNS tunneling place non-DNS traffic within DNS traffic to circumvents(ËßÑÈÅø) security solutions,and the different types of DNS records such as TXT,MX,SRV,NULL,A or CNAME are altered.

    DHCP Attack

    Could result in every host on the network communication with malicious DNS servers and gateways.A DHCP spoofing attack creates a rouge DHCP server to serve falsified information.

Enterprise Servicesüëá

HTTP & HTTPS

  1. Malicious iFrames

    An iFrame allows a page from a different domain to be opened inline within the current page.It can be used to launch malicious code.

  2. HTTP 302 cushioning

    Allows a web page to redirect and open in a different URL.Can be used to redirected to malicious code.

  3. Domain shadowing

    Malicious web sites are created from subdomains created from a hijacked domain.

    Email

  4. Attachment-based attacks

    Email with malicious executable files attached.

  5. Email Spoofing

    Phishing(ÁΩëÁªúËØ±È™óÔºâattack where the message appears to come from a legitimate source.

  6. Spam email

    Unsolicited email with advertisements or malicious content.

  7. Open mail relay server

    Massive amount of spam and worms can be sent by misconfigured email servers.

  8. Homoglyphs

    Phishing scheme where text characters (hyperlinks) look similar to real text and links.

    Web-Exposed Databases

  9. Command injection attacks

    Insecure code and web applications allows OS commands to be injected into form fields or the address bar.

  10. XSS Cross-site scripting attacks

    Insecure server-side scripting where the input is not validated allows scripting commands to be inserted into user generated forms fields, like web page comments.This results in visitors being redirected to a malicious website with malware code.

  11. SQL injection attacks

    Insecure server-side scripting allows SQL commands to be inserted into form fields where the input is not validated.

  12. HTTP injection attacks

    Manipulation of html allows executable code to be injected through HTML div tags, etc.
